A Short History of Testing of Software Systems

Santiago Vera Espinoza
3 mins read - September 2, 2023

One of the things I come to think about from time to time since I decided that I wanted to become a software engineer is a video that was shown to me during the first semesters of my career. The video talked about the huge responsibilities that we have as developers: our code is used everyday and a lot of people depend on the availability of what we do.

For example, I work as a software engineer intern in Oracle, more specifically in the devops area of the database team. I, in fact, develop test systems for the big database servers that hundreds of companies use on a daily basis. For example, banks, assurance companies, research organizations (like CERN), video game companies (Activision), the famous Oracle RedBull racing team and the US army. This info is public by the way, you can check it here , here and here (I swear I'm not breaking any privacy agreement here, please don’t kill me Oracle).

Knowing this you can imagine just how important it is for systems to be highly available and tested so users can use (jajaja, funny wording there) the services provided and these services tend to be vital. Imagine if a known bug that could make the bank system of a big company was not detected by Oracle’s Exachk system and made the server crash, catastrophe! And believe me these are common issues.

It is really important to test systems so we don’t get these types of errors. We have a great responsibility in our hands. So in this case I’m going to honor this area of software development by making a quick chronology of testing in software systems, get ready!

So let’s go through it right away:

  • 1950: Tom Kilburn creates the first software for the University of Manchester in England. It performed mathematical calculations using machine code instructions.
  • Again in 1950: The Turing test was introduced this year which checks the intelligence of machines.
  • 1957: Debugging started being considered as a software testing method. This was made via news articles and the experience of various developers.
  • 1958: Gerald M. Weinberg forms the first team specifically focused on testing systems.
  • 1961: A specific chapter on software testing was introduced in the book “Computer Programming Fundamentals” by Gerald Weinberg & Herbert Leeds.
  • 1968: Software testing was mentioned in a report by NATO.
  • 1970: Winston Royce described a waterfall model in the paper Managing the Development of Large Software Systems.
  • 1971: Richard Lipton proposed the concept of mutation testing. Mutation Testing is a type of software testing in which certain statements of the source code are changed/mutated to check if the test cases are able to find errors in the source code.
  • 1973: Cause Effect Graphing-based technique was introduced.
  • 1978: Functional testing was introduced by William Howden.
  • 1979: Destruction oriented era. In this era, the focus was to break the code and find some errors. That’s why it is called like that!
  • 1983: The first version of the IEEE 829 Standard for Software Test Documentation was published.
  • 1985: Autotester, the first commercial test tool for the PC is released.
  • 1986: Software quality engineering was founded by David Gelperin and William Hetzel.
  • 1988-2000: Testing had reached a qualitatively new level, which led to the further development of methodologies, and powerful tools for managing the testing process.
  • 2004: Selenium, a web application testing tool was developed.
  • 2005: SoapUI was released in the month of September and it became quite popular.
  • 2008: Software testing company Altom is founded.
  • 2016: Pytest gets formally released.
  • Present: Artificial intelligence is also being used for testing software based on the behavior of the end users. Now testers perform different types of testing like Unit testing, Integration testing, Acceptance testing, Performance testing, etc.

This sure was a big trip indeed. We should stop by sometimes and think about everything that has brought us to the point we are now. How such an important thing as testing was evoluted until this day.
